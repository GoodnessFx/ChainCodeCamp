01

The Foundation

1 Week Duration

Before writing a single line of Solidity, you must internalize the fundamental constraints of a decentralized environment. Smart contracts are immutable, transparent, and high-stakes. There is no "undo" button in production.

Public-Key Cryptography

The bedrock of blockchain identity. You must understand that an Ethereum address is simply the result of a cryptographic pipeline:

  1. Generate a Private Key (256-bit random number).
  2. Derive the Public Key via ECDSA.
  3. Hash the Public Key using keccak256.
  4. Take the last 20 bytes of that hash.

Security Mindset #1

Identity is ownership. If a private key is compromised, the assets are gone. As a developer, you never handle user private keys directly; you design systems that interact with signatures.

02

EVM Internals

2 Weeks Duration

The Ethereum Virtual Machine is a stack-based state machine. To be an elite developer, you must think in terms of opcodes and gas costs, not just high-level Solidity syntax.

Location Gas Cost Nature Security Context
Stack Minimal (3) Volatile 1024 slots. Source of "Stack Too Deep" errors.
Memory Quadratic Volatile Cheap for small data, exponentially expensive for large.
Storage Expensive (20k+) Permanent The state of the world. Every SLOAD/SSTORE must be optimized.

The DelegateCall Pattern

One of the most powerful and dangerous opcodes in the EVM. It allows a contract to execute code from another contract but within its own storage context.

vulnerability_example.sol 
// DANGER: Malicious contract can overwrite owner slot
(bool success, ) = target.delegatecall(
    abi.encodeWithSignature("attack()")
);
03

Solidity & Yul Mastery

4 Weeks Duration

Solidity is the high-level language, but Yul is the assembly language that gives you surgical control over the EVM. Elite auditors look for patterns that high-level compilers might miss.

Advanced Patterns

  • Bitwise Operations: Packing multiple variables into a single 32-byte storage slot to save gas.
  • Assembly Optimization: Using mload and mstore for efficient data handling.
  • Custom Errors: Using error Name() instead of strings to save gas on revert.
04

Security & Exploits

Continuous Deep Dive

Security is not a stage; it is the entire process. You must learn to think like an attacker to build as a defender. Every major DeFi hack in history is a lesson in code logic.

Case Study: Reentrancy

The exploit that birthed Ethereum Classic. An attacker calls a function that makes an external call to the attacker's contract, which then calls back into the original function before the state is updated.

secure_withdrawal.sol 
function withdraw() public {
    uint balance = balances[msg.sender];
    require(balance > 0);
    
    // 1. Update state FIRST (Checks-Effects-Interactions)
    balances[msg.sender] = 0;
    
    // 2. Then interact
    (bool success, ) = msg.sender.call{value: balance}("");
    require(success);
}
